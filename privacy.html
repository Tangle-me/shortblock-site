Privacy Policy

Last updated: May 2026

Summary

ShortBlock does not collect, transmit, or store any personal data on any server. Everything the extension stores stays on your device. We have no servers that receive your data.

What we store — and where

The ShortBlock Chrome extension stores two things locally in your browser using chrome.storage.local:

  • A hashed PIN. When you set a parent PIN, it is hashed using a one-way function and stored in your browser's local storage. The original PIN is never stored. This data never leaves your device.
  • A protection state flag (on or off). This is a single boolean value stored locally.

Neither value is ever transmitted to ShortBlock or any third party.

Why the extension accesses certain websites

The extension uses Chrome's declarativeNetRequest API and a content script to block short-form video. Access to the following domains is required only to apply blocking rules — no data from those sites is ever read or transmitted:

  • tiktok.com, tiktokv.com, tiktokcdn.com, musical.ly — to block TikTok and its media/API domains
  • youtube.com — to detect and redirect navigation to youtube.com/shorts
  • instagram.com, cdninstagram.com — to block Instagram Reels media
  • snapchat.com — to block Snapchat Spotlight pages

No tracking, no analytics

ShortBlock contains no analytics, no telemetry, no crash reporting, and no advertising. We do not know who installs the extension or how it is used.

No account required

The Chrome extension works entirely without an account. The optional Parent Dashboard on this website uses Supabase for authentication. If you create an account, your email address and content-blocking preferences are stored in our Supabase database. You can request deletion at any time by emailing us.

Children's privacy

ShortBlock is a tool for parents to protect children. We do not collect data from children. The extension does not interact with any child's data — it only blocks certain network requests on the device where it is installed.

Changes to this policy

If we update this policy, we will change the "Last updated" date above. Continued use of ShortBlock after changes constitutes acceptance of the updated policy.
